# Bob AI Studio

Android-редактор в духе CapCut: импорт видео (например, эдита из TikTok),
удаление водяного знака, замена фрагментов таймлайна ("Change Edit") с
ИИ-подписями сцен через Groq, и аудиодорожка с расстановкой битов и
магнитным прилипанием фрагментов к ритму.

## Сборка APK

### Через GitHub Actions (рекомендуется)

1. Залей этот репозиторий на GitHub.
2. Зайди в **Settings → Secrets and variables → Actions → New repository secret**
   и добавь секрет `GROQ_API_KEY` со своим ключом с [console.groq.com](https://console.groq.com).
3. (Необязательно, для подписанного release-APK) добавь секреты:
   `SIGNING_STORE_FILE`, `SIGNING_STORE_PASSWORD`, `SIGNING_KEY_ALIAS`, `SIGNING_KEY_PASSWORD`.
   Без них сборка release всё равно пройдёт — просто будет подписана debug-ключом.
4. Пушни в ветку `main` или запусти workflow вручную (**Actions → Build Bob AI Studio APK → Run workflow**).
5. Готовые APK появятся в разделе **Actions → (последний запуск) → Artifacts**:
   `bob-ai-studio-debug` и `bob-ai-studio-release`.

### Локально

```
export GROQ_API_KEY=твой_ключ
./gradlew assembleDebug
```
APK появится в `app/build/outputs/apk/debug/app-debug.apk`.

## Что реально работает в этой версии (MVP)

- **Remove Watermark** — ты вручную обводишь прямоугольник вокруг водяного
  знака (как в CapCut), приложение прогоняет видео через FFmpeg-фильтр
  `delogo`, который "закрашивает" эту зону по всем кадрам, беря пиксели с
  краёв области. Работает надёжно для полупрозрачных ников авторов TikTok.
  Остальная часть кадра (и любые эффекты вроде глитча) не трогается.
- **Change Edit** — таймлайн режется на фрагменты, для каждого берётся
  опорный кадр, который отправляется в Groq (модель `qwen/qwen3.6-27b`,
  с фолбэком на `meta-llama/llama-4-scout-17b-16e-instruct`) — модель
  возвращает короткое описание сцены. Дальше показывается сетка фрагментов;
  тап по фрагменту заменяет его твоим фото/видео из галереи, при этом
  эффекты, слот на таймлайне и синхронизация с битами сохраняются. Фото
  умеет растягиваться по времени лёгким Ken Burns-зумом.
- **Аудиодорожка** — визуализация громкости (waveform), режим расстановки
  битов (тап — добавить точку, тап по точке — убрать), магнитное
  прилипание края фрагмента к ближайшему биту (±150мс), которое
  автоматически "отпускает", если тянуть дальше этого окна.

## Важные технические примечания

- **FFmpegKit официально снят с поддержки** (бинарники убраны с Maven
  Central 1 апреля 2025). Проект использует `io.github.xch168:ffmpeg-kit-full-gpl`
  — актуальный на 2026 год форк с тем же API. Если этот координат тоже
  устареет, замени зависимость в `app/build.gradle.kts` на актуальный форк —
  весь остальной код (`FFmpegKit.executeAsync`, `ReturnCode` и т.д.) менять
  не придётся, так как API идентичен оригинальному FFmpegKit.
- **"Удаление водяного знака через ИИ, распознающий текст и картинку"**:
  честно — Groq не хостит модели для video inpainting (закрашивания
  движущихся объектов на видео), только текстовые/vision LLM. Поэтому в
  этой версии зона выделяется вручную, а сам "ремонт" пикселей делает
  FFmpeg. Groq используется по назначению — для распознавания *содержимого*
  кадров (Change Edit) и для потенциального автоопределения зон текста
  (`SceneAnalyzer` уже возвращает `text_boxes`, это можно подключить как
  подсказку для авто-выделения зоны в следующей версии).
- Ключ Groq зашивается в `BuildConfig.GROQ_API_KEY` на этапе сборки — не
  хранится в коде репозитория.

## Структура проекта

```
app/src/main/java/com/bobai/studio/
├── ai/          — Groq API клиент и анализ сцен
├── audio/       — waveform + детекция/магнит битов
├── data/        — модели проекта, Room DAO
├── editor/      — WatermarkRemover, FragmentReplacer, EditorViewModel
└── ui/          — экраны Compose (Home, Editor, Change Edit, таймлайн, аудиодорожка)
```
